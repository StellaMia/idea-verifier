# 想法验证器（Idea Verifier）

对网络内容进行工程级审计的标准化测试框架。帮你快速判断项目、工具、文章、事件的真实性、可行性与风险，避免踩坑。

---

## 快速开始

将 `skill.md` 的全部内容复制到以下任意位置使用：

| 使用方式 | 操作 |
|----------|------|
| **Claude.ai Project** | 粘贴到 Project Instructions 中，之后直接发链接即可触发 |
| **Claude Code Skill** | 保存为 `.claude/commands/idea-test.md`，通过 `/idea-test` 触发 |
| **其他 AI 平台** | 作为 System Prompt 粘贴，每次对话时发送链接或内容 |

---

## 使用方法

1. 触发 Skill 后，直接发送你想测试的内容：
   - 公众号 / 小红书 / B站 / 知乎链接
   - 直接粘贴文章全文
   - 简要描述一个项目或想法
2. Skill 自动识别内容类型，选择对应的测试维度
3. 输出标准化报告，包含各维度评分与综合结论

---

## 测试维度

| # | 维度 | 适用类型 |
|---|------|----------|
| 1 | 真实性校验 | 全部 |
| 2 | 批判性解读 | 全部 |
| 3 | 逻辑可行性 | 全部 |
| 4 | GitHub 硬核核查 | 技术 / 工具类 |
| 5 | 部署与可复现性 | 技术 / 工具类 |
| 6 | 风险点识别 | 全部 |
| 7 | 水分度判断 | 全部 |
| 8 | 内容定级与分类 | 全部 |
| 9 | 综合结论 | 全部 |

---

## 注意事项

- **水分度评分**：越低越真实（1 = 零水分，5 = 严重水分）
- **风险维度评分**：越低风险越高（5 = 无风险，1 = 高风险）
- 无法访问的链接，Skill 会基于现有信息部分执行，并在报告顶部标注

## 文件结构

```

├── skill.md              # 主 Skill 文件（可直接分发使用）
├── README.md             # 本文件
├── rubrics/

├── examples/
│   ├── tech-project.md   # 案例：开源技术项目测试
│   └── event.md          # 案例：维权事件测试
└── CHANGELOG.md          # 版本记录
```

## 适用场景

- 评估网络上的项目 / 工具是否值得跟进
- 判断文章 / 案例的水分与风险
- 快速了解新工具 / 新想法的可行性
- 识别网络侵权、维权事件的真实性
- 为独立开发者 / 从业者提供项目参考
