# 评分细则参考手册

本文档是「想法验证器」各维度评分标准的详细说明，供迭代维护参考。

---

## 评分方向说明

大多数维度：**评分越高越好**（5 = 最佳，1 = 最差）

以下两个维度方向相反，注意区分：

| 维度 | 方向 | 说明 |
|------|------|------|
| 水分度（维度 7）| 越低越好 | 1 = 零水分，5 = 严重水分 |
| 风险识别（维度 6）| 越低风险越高 | 5 = 无风险，1 = 高风险 |

---

## 维度 1 — 真实性校验

**核心问题**：内容的核心事实能否被独立验证？

| 分值 | 判断标准 | 典型信号 |
|------|----------|----------|
| 5 | 核心事实有多个独立来源交叉验证，数据可追溯 | 官方公告 + 第三方媒体 + 数据工具截图 |
| 4 | 核心事实可信，少量细节无法验证 | 主要来源可信，细节数字未附来源 |
| 3 | 核心事实可信，但关键数据缺乏来源 | 「据说」「业内人士透露」等模糊表述 |
| 2 | 存在无法核实的核心声明 | 单一来源、自我背书、不可查证的数据 |
| 1 | 存在明显虚构 / 捏造 | 与已知事实矛盾，或事后被证伪 |

**标题党识别要点**：
- 标题使用「震惊」「突破」「颠覆」等词，但正文未有对应内容
- 数字夸大：标题「45 万访问」，正文未提供时间段或来源
- 主语偷换：标题主语与内容主语不同

---

## 维度 2 — 批判性解读

**核心问题**：作者想让你相信什么？这个论点成立吗？

| 分值 | 判断标准 |
|------|----------|
| 5 | 论点有据、边界清晰、主动说明局限性与反例 |
| 4 | 论点基本成立，有少量过度推断但不影响主旨 |
| 3 | 论点可信但边界模糊，部分前提假设未声明 |
| 2 | 存在明显逻辑跳跃，或隐藏了重要的利益关系 |
| 1 | 核心论点站不住脚，或存在明显误导意图 |

**常见隐藏前提**：
- 「A 产品做到了 X，所以你也能做到」（忽略资源差异）
- 「我的方法让我月入百万」（忽略幸存者偏差）
- 「行业趋势是 X，所以应该做 Y」（趋势推断过于武断）

---

## 维度 3 — 逻辑可行性

**核心问题**：如果按照内容所描述的路径执行，逻辑上能走通吗？

| 分值 | 判断标准 |
|------|----------|
| 5 | 逻辑链完整，无漏洞，结论有充分支撑 |
| 4 | 逻辑基本成立，有小漏洞但不影响结论 |
| 3 | 整体逻辑可信，但存在需要额外假设才能成立的环节 |
| 2 | 存在明显逻辑漏洞，结论过度跳跃 |
| 1 | 逻辑链断裂，结论无法从前提推出 |

**常见逻辑漏洞**：
- 循环论证：「这个产品好，因为大家都说好」
- 假关联：「用了 A 之后收入增加，所以 A 导致收入增加」
- 排除中间项：「要么大成功，要么失败」（忽略中间状态）

## 维度 4 — GitHub 硬核三维核查

**适用范围**：所有声称开源或可部署的技术 / 工具类内容

| 分值 | 判断标准 |
|------|----------|
| 5 | 完整开源，代码可审计，有完整文档，CI/CD 健全，一键部署 |
| 4 | 开源但文档不完整，部署需要一定门槛，代码质量可接受 |
| 3 | 部分开源或代码可见，存在不透明组件或过时依赖 |
| 2 | 声称开源但仓库空 / 私有 / 代码混乱，无法实际使用 |
| 1 | 无法验证，代码不可信，或存在明显安全隐患（硬编码密钥等）|

**核查清单**：
- [ ] 仓库是否存在（github.com/xxx 可访问）
- [ ] 最近 commit 时间（超过 1 年未更新为黄色信号）
- [ ] Star 数与宣传流量是否匹配（差距过大需注意）
- [ ] README 是否有安装 / 部署说明
- [ ] 依赖是否来自可信来源（npm、PyPI 等）
- [ ] Issues 是否有活跃响应

---

## 维度 5 — 部署与可复现性

**适用范围**：技术 / 工具类内容

| 分值 | 判断标准 |
|------|----------|
| 5 | 零门槛，普通人 30 分钟内可独立复现，无隐藏成本 |
| 4 | 低门槛，有基础技术能力（会用命令行）即可，费用透明 |
| 3 | 中等门槛，需要特定环境、付费 API 或较强技术能力 |
| 2 | 高门槛，实际操作比宣称复杂得多，或存在隐藏费用 |
| 1 | 无法复现，依赖不可控的私有资源或未公开条件 |

**复现成本清单**：
- 算力需求（本地 GPU？云服务费用？）
- API 费用（OpenAI、Anthropic 等按量计费）
- 账号要求（需要特定地区？需要付费订阅？）
- 时间成本（配置环境通常比演示视频长 5–10 倍）

## 维度 6 — 风险点识别

**注意**：此维度评分越低代表风险越高

| 分值 | 判断标准 |
|------|----------|
| 5 | 无明显风险，内容中性客观，无诱导信号 |
| 4 | 有轻微风险信号，但影响有限，读者容易识别 |
| 3 | 存在中等风险，需要读者具备一定鉴别能力 |
| 2 | 风险较高，存在明显诱导或重要信息未披露 |
| 1 | 高风险，存在欺骗性内容、严重利益冲突或操控性设计 |

**风险信号清单**：
- 诱导性：限时、稀缺、紧迫感制造（「名额有限」「今天最后一天」）
- 权威背书：无法验证的专家 / 机构背书
- 变现陷阱：免费内容引导付费，付费内容价值不透明
- 依赖风险：核心功能依赖可能随时消失的第三方服务
- 利益冲突：测评者与被测评方有商业关系但未披露

## 维度 7 — 水分度判断

**注意**：此维度评分越低越真实

| 分值 | 判断标准 | 典型表现 |
|------|----------|----------|
| 1 | 零水分 | 标题克制，数据有来源，无情绪化用语 |
| 2 | 轻微水分 | 标题稍有引导，核心内容与标题基本一致 |
| 3 | 中等水分 | 有夸大但能找到数据支撑，情绪化表达明显 |
| 4 | 较多水分 | 核心事实真实但被大量渲染包装 |
| 5 | 严重水分 | 内容与标题严重不符，主要靠情绪和悬念吸引 |

**水分信号词**：
- 「颠覆」「革命性」「史上最强」「绝对」
- 「我试了 30 天，结果震惊了」
- 「99% 的人都不知道」
- 数字精确到不合理的程度（「访问量精确到个位数」）

---

## 维度 8 — 内容定级与分类

**不评分，输出描述性标签**

内容类型参考：
- 工具测评：第一方或第三方使用体验分享
- 事件记录：真实发生的事件陈述
- 观点分析：对行业 / 现象的主观判断
- 案例复盘：对已有结果的回顾分析
- 教程指南：操作步骤或方法论分享

深度等级：
- 入门：无背景知识即可理解
- 中级：需要一定领域基础
- 专业：面向从业者，包含专业术语和深度分析

---

## 综合结论 — 风险等级判定规则

根据所有维度评分，按以下规则判定总体风险等级：

| 风险等级 | 判定条件 |
|----------|----------|
| 🟢 低 | 真实性 ≥ 4，风险识别 ≥ 4，水分度 ≤ 2，无维度低于 3 |
| 🟡 中 | 存在 1–2 个维度评分为 2–3，或水分度为 3–4 |
| 🔴 高 | 任一维度评分为 1，或风险识别 ≤ 2，或真实性 ≤ 2 |

推荐行动参考：
- 🟢 低风险 → 收藏参考，可放心传播
- 🟡 中风险 → 谨慎对待，注明局限性后可有选择地分享
- 🔴 高风险 → 不建议传播，说明具体原因
